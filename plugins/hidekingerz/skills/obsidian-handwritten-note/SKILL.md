# 手書き手帳メモ → Obsidian Markdown 変換スキル

## 概要

手帳に手書きされたメモの**写真・画像**を受け取り、Obsidianで使えるMarkdownファイルとして整形・出力する。
主なユースケースは**日々のタスク・TODO管理**と**アイデア・ブレスト**。

---

## ステップ

### 1. 画像を丁寧に読み取る

- 手書き文字を可能な限り正確に読み取る
- **読み取れない箇所**は `[読み取り不可]` とマークする
- 走り書き・略語は文脈から推測して補完してよい（補完した場合は末尾の変換メモに記載）
- 視覚的な構造（矢印、囲み、段組み、インデント）もMarkdownの構造に反映する
- 日付・曜日の記載があれば必ず拾う

#### 不明点はユーザーに確認する

読み取り結果に自信が持てない場合や、解釈が複数考えられる場合は、**勝手に判断せずユーザーに質問する**。
たとえば以下のようなケースでは確認を取ること：

- 文字が潰れていて複数の読み方ができる（例：「7」か「1」か、「ア」か「マ」か）
- 略語・専門用語で意味が不明なもの
- 取り消し線があるが完了なのか削除なのか判断がつかない
- 日付が曖昧（例：数字だけで年月日のどれか分からない）

無理に推測して間違えるより、ユーザーに聞いたほうが結果の品質が上がる。
ただし明らかに文脈から分かるもの（例：「Todo」の下に書いてある項目はタスク）まで確認する必要はない。
**「聞くべきか迷ったら聞く」** くらいのバランスで。

---

### 2. デイリーノートか専用ノートかを判断する

**デイリーノート形式**を使う場合（以下に当てはまるとき）：
- 日付が書かれている
- 「今日やること」「今日の気づき」など、その日の記録としての性格が強い
- タスクとアイデアが混在している

**専用ノート形式**を使う場合：
- 特定テーマに絞った深い内容（企画書、調査メモなど）
- 日付より「トピック」が主役の場合

---

### 3. YAML フロントマターを生成する

```yaml
---
date: YYYY-MM-DD        # 手帳の日付（画像から読み取る。不明なら今日の日付）
tags: []                # 内容から推測（3〜5個。例: tasks, idea, guitar, work）
---
```

**シンプルに保つ。** フィールドはこの2つで十分。タグは内容のテーマを反映する英単語か日本語で。

---

### 4. 本文をMarkdownに変換する

#### デイリーノートの構造（推奨テンプレート）

```markdown
# YYYY-MM-DD（曜日）

## 📋 タスク

- [ ] タスク1
- [ ] タスク2
- [x] 完了済みタスク

## 💡 アイデア・気づき

> [!idea]
> アイデアの内容

- 気づいたこと
- 思いついたこと

## 📝 メモ

その他のメモ・雑記
```

セクションは**内容があるものだけ**出力する。空のセクションは省略。

変換ルールの詳細（セクション自動判別、見出し構造、タスク変換、コールアウト）は [reference.md](reference.md) を参照。

---

### 5. ファイル名を決める

**デイリーノート：**
```
YYYY-MM-DD.md
```

**専用ノート：**
```
YYYY-MM-DD トピック名.md
```

- 日付は手帳から読み取った日付（不明なら今日）
- トピック名は10〜20字、日本語OK、記号・スラッシュは除く

---

### 6. ファイルの配置先

生成したMarkdownファイルはObsidian Vault内の適切なディレクトリに保存する。

**デイリーノート：**
```
daily-notes/YYYY/MM/YYYY-MM-DD.md
```
例: `daily-notes/2026/03/2026-03-14.md`

年・月のサブフォルダが存在しない場合は自動で作成する。

**専用ノート：**
Vault直下、またはトピックに応じた既存フォルダに配置する。

---

### 7. 出力する

以下の順で出力する：

1. **ファイル名**（コードブロック外に `**ファイル名:** \`...\`` 形式で明示）
2. **Markdownの内容**（コードブロック内）
3. **変換メモ**（読み取れなかった箇所・補完した内容がある場合のみ記載）

### 8. 完了を確認する

ファイル書き出し後、以下を実行して正しく作成されたことを確認する：

- ファイルが指定パスに存在するか確認する
- フロントマターのYAML構文が正しいか確認する

---

## 出力例

**ファイル名:** `2025-03-12.md`

````markdown
---
date: 2025-03-12
tags: [tasks, guitar, work]
---

# 2025-03-12（水）

## 📋 タスク

- [x] Morning Glory 試奏
- [ ] React Query ドキュメント読む
- [ ] ペダルボード ケーブル注文
- [ ] ボードサイズ測定

## 💡 アイデア・気づき

> [!idea]
> Morning Glory → クリーンブースターとしても使えそう。OD的な使い方もいける。

> [!tip]
> [[Strymon Zuma R300]] は Dream '65 に9V 300mA以上必要。購入前に確認。
````

---

## 注意事項

- **原文の意図を尊重する**。内容を勝手に要約・変更しない
- 読み取りに自信がない箇所は `[読み取り不可]` または `[推測: ...]` とマークする
- 補完した情報は末尾の変換メモに記載する
- 日本語のメモは日本語のまま出力する
- [[内部リンク]] は固有名詞（ツール名・プロジェクト名）に使う。やりすぎない
